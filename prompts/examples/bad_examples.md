# Bad Examples — Anti-Patterns to Reject

These are examples of content that should NEVER be produced by the generator. They represent the most common failure modes. The reviewer uses these to flag content that has drifted from the brand.

---

## Anti-Pattern 1: Generic Motivation Content
```
BAD:
Consistency is the key to growth.

Show up every day. Build the habit. Trust the process.

Your future self will thank you. 🙏
```
**Why it fails:** Could have been written by anyone, for anyone. No specific claim. No technical anchor. Pure filler. The emoji confirms it.

---

## Anti-Pattern 2: Hype Language
```
BAD:
This new AI tool is absolutely game-changing for developers.

I've been testing it all week and the results are insane. This is going to revolutionize how we build.

You NEED to try this. 🔥🔥🔥
```
**Why it fails:** Every banned word is present. No specifics. No critical angle. Reads like a sponsored post even if it isn't. Destroys brand credibility.

---

## Anti-Pattern 3: Tutorial Voice
```
BAD:
Here are 5 ways to improve your code quality:

1. Write clean functions
2. Use meaningful variable names
3. Add comments
4. Write tests
5. Refactor regularly

Which one do you struggle with most? Drop a comment! 👇
```
**Why it fails:** Generic tip list. The CTA is engagement bait. Could have been posted by any dev account in 2019. No specific insight. No Dave voice.

---

## Anti-Pattern 4: Performed Humility
```
BAD:
Just a dev figuring things out one bug at a time.

Sharing my journey in case it helps someone else. Not an expert — just building in public and learning as I go. 🌱
```
**Why it fails:** False modesty undermines brand authority. Dave is not "just a dev." He's a skilled builder with specific expertise. This voice pattern is common and indistinguishable from thousands of other accounts.

---

## Anti-Pattern 5: The Fabricated Experience
```
BAD:
I shipped this feature and got 500 signups in 24 hours.

Here's the exact workflow I used:
[Generic workflow that could apply to any project]
```
**Why it fails:** Implies specific metrics (500 signups) that weren't in the source material. This is fabrication. The brand rule is clear: no invented metrics.

---

## Anti-Pattern 6: Empty Thread Padding
```
BAD:
Tweet 1: I want to talk about something important for developers. A thread.

Tweet 2: First, let me set the context for why this matters.

Tweet 3: With that background established, let's get into it.

Tweet 4: Here's the first thing I want to cover:
[Actual content finally appears in tweet 5]
```
**Why it fails:** Three setup tweets before the content starts. This is the structural equivalent of a slow article intro. On X, if tweet 1 isn't the hook, the thread fails.

---

## Anti-Pattern 7: Corporate Voice
```
BAD:
In today's rapidly evolving tech landscape, developers face unprecedented challenges when it comes to shipping quality products at scale.

It's important to note that the most successful development teams prioritize a holistic approach to their workflow.
```
**Why it fails:** Sounds like a LinkedIn article written by committee. Every phrase is a red flag: "rapidly evolving tech landscape," "unprecedented challenges," "it's important to note." Delete on sight.

---

## Anti-Pattern 8: Hashtag Spam
```
BAD:
Just shipped the new auth module. Feeling good about this one.

#developer #coding #buildinpublic #javascript #webdev #indie #solodev #programming #100daysofcode #shipit
```
**Why it fails:** Ten hashtags. The content is fine. The hashtags transform it into content-machine output. Maximum one hashtag, only when it meaningfully categorizes the post in a specific community.

---

## Anti-Pattern 9: The Non-Contrarian Contrarian
```
BAD:
Unpopular opinion: you should write good code.

Most developers skip this step and it costs them later. Good code = fewer bugs, faster iteration, happier team.

RT if you agree.
```
**Why it fails:** "Unpopular opinion" signals a contrarian post, but the actual content is completely mainstream. The "RT if you agree" is engagement bait. The whole post is a non-take dressed up as a take.

---

## Anti-Pattern 10: Vague Failure Story
```
BAD:
I failed at my side project.

It was hard. I learned a lot. But I'm glad I went through it because it made me stronger as a developer.

Never give up. 💪
```
**Why it fails:** No specifics about what failed, why it failed, or what was actually learned. The lesson ("never give up") is a cliché. The emoji confirms the worst. A real failure story includes the specific failure.
